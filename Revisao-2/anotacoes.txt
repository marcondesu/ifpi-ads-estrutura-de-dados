.Para remover o último elemento de uma LISTA SIMPLESMENTE ENCADEADA, apesar
de já sabermos quem é este elemento, é necessário percorrer os n elementos da lista e
parar a frente do último elemento para só então poder removê-lo, pois não existe acesso
imediato aos seus predecessores. Portanto, o algoritmo para remover o último nó da lista
possui complexidade O(n).

.Em uma LISTA DUPLAMENTE ENCADEADA, sabendo-se que a acessibilidade
do último nó é imediata, o algoritmo para remover o último nó da lista é executado em
tempo constante O(1).

.Um exemplo de sua aplicação é quando diversos processos estão usando os
mesmos recursos ao mesmo tempo e temos de assegurar que nenhum processo acesse
o recurso antes de todos os outros processos. Os processos são colocados em uma
LISTA CIRCULAR, acessíveis através do ponteiro "corrente". Depois que
um nó da lista é acessado e o número do processo é recuperado do nó para ativar esse
processo, "corrente" se move para o próximo nó de modo que o próximo processo possa
ser ativado a seguir.

Uma PILHA é uma estrutura linear de dados que pode ser acessada
somente por uma de suas extremidades para armazenar e recuperar dados.

Uma FILA é simplismente uma linha de espera que cresce somando
elementos ao seu final e que diminui tomando elementos de sua frente, desta forma,
ambas as extremidades são utilizadas.

Busca linear
    Verifica valor por valor, do primeiro ao último para checar.
    Vantagem - funciona mesmo quando o vetor não está ordenado  
    Melhor caso - quando o valor procurado está na primeira posição
    Pior caso - complexidade de tempo O(n)
